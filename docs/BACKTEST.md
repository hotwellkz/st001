# Backtester — архитектура и честность

**Исторический бэктест и команды:** см. [BACKTEST-HISTORICAL.md](./BACKTEST-HISTORICAL.md).

## Архитектура

1. **Исторические свечи** — `HistoricalCandleSource`: по символу массив `Candle` по времени (только закрытые).
2. **Симулятор** — дискретные шаги по индексу `i` (все символы **выровнены** по одному и тому же списку `closeTime`).
3. **Решение на баре i** — только `candles[0..i]`; сигнал = домен `evaluateEntryAtIndex` / `evaluateExitLongAtIndex`.
4. **Исполнение** — **следующая** свеча `i+1`: вход/выход по **open** с проскальзыванием + комиссия taker на оборот.
5. **Портфель** — единый cash, позиции по символам, лимит heat и max concurrent из `StrategyMvpConfig`.
6. **Метрики** — после прогона; walk-forward — нарезка окон по индексам.

## Где бэктест может «врать» (явно)

| Источник | Риск |
|----------|------|
| **Исполнение по open следующей свечи** | Реальный MARKET может исполниться хуже (гэпы, очередь). |
| **Фиксированный slippage %** | Волатильные дни дают больший проскал; мы недооцениваем хвост. |
| **Нет partial fills** | Крупные ордера на illiquid парах — иное исполнение. |
| **Идеальные high/low стопа** | Стоп по close бара — не intrabar stop hunt. |
| **Синхронные бары по всем символам** | Пропуски тиков/разная ликвидность не моделируются. |
| **Survivorship** | Universe только «живые» монеты — завышение прошлого. |
| **Walk-forward без учёта смены параметров** | Если подгонять параметры на in-sample — out-of-sample всё равно оптимистичен. |

## Допущения

- Long-only, та же логика, что live/paper (домен).
- Комиссия симметрична на вход и выход (taker).
- Размер позиции от **equity на close бара i** (чуть оптимистично vs equity на open i+1 — см. ограничение ниже).

## Чеклист валидации

- [ ] Сигнал на индексе i не использует `candles[i+1]`.
- [ ] Цена входа = `open[i+1] * (1 + slipBuy)`; выход = `open[i+1] * (1 - slipSell)`.
- [ ] При отсутствии бара `i+1` сделка не открывается в этом шаге.
- [ ] Heat и max positions проверяются до входа.
- [ ] Один и тот же прогон даёт детерминированный результат.
- [ ] Эквити никогда не использует будущие цены для прошлых решений.

## Ограничения MVP

- Выравнивание баров: все символы должны иметь одинаковую длину и те же `closeTime` (или runner пропускает шаг).
- Нет funding, нет borrow.
